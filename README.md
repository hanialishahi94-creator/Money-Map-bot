# Money MAP — بات تلگرام + پنل مدیریت

این پروژه از دو بخش تشکیل شده:

- **`moneymap_bot/`** — بات تلگرام (`telegram_bot.py`) و بک‌اند پنل مدیریت (`admin_panel.py`، یک REST/JSON API روی Flask) به همراه دیتابیس SQLite مشترک (`db.py`).
- **`minimapa/`** — فرانت‌اند پنل مدیریت (React + TypeScript + Vite + Tailwind)، با برندینگ **Money MAP**، که از طریق `/api/*` با بک‌اند Flask صحبت می‌کند.

## توسعه (Development)

برای کار روی پنل به‌صورت لوکال، دو سرویس را به‌صورت همزمان بالا بیاور:

### ۱. بک‌اند Flask (API)

```bash
cd moneymap_bot
pip install flask
python admin_panel.py
```

به‌صورت پیش‌فرض روی پورت `8000` بالا می‌آید (با متغیر محیطی `PORT` قابل تغییر است).

متغیرهای محیطی مهم:

| متغیر | توضیح | پیش‌فرض |
|---|---|---|
| `SECRET_KEY` | کلید رمزنگاری session فلسک | `moneymap-secret-key-change-me` |
| `ADMIN_USERNAME` | نام کاربری ورود به پنل | `09124900216` |
| `ADMIN_PASSWORD` | رمز عبور ورود به پنل | `Saba09124900216` |
| `BOT_TOKEN` | توکن بات تلگرام (برای ارسال پیام گروهی از پنل) | — |
| `DB_PATH` | مسیر فایل دیتابیس SQLite | `/data/moneymap.db` در صورت وجود پوشه `/data`، در غیر این صورت `moneymap.db` |

> پیشنهاد می‌شود `ADMIN_PASSWORD` و `SECRET_KEY` را روی هر دیپلوی واقعی حتماً از طریق متغیر محیطی override کنید و مقدار پیش‌فرض کد را برای پروداکشن استفاده نکنید.

### ۲. فرانت‌اند React (پنل مدیریت)

```bash
cd minimapa
npm install
npm run dev
```

سرور Vite روی پورت `5173` بالا می‌آید و طبق تنظیمات `vite.config.ts`، تمام درخواست‌های `/api/*` به‌صورت خودکار به `http://127.0.0.1:8000` (همان Flask) پروکسی می‌شوند — یعنی هیچ مشکل CORS‌ای در حالت توسعه وجود ندارد.

پنل را در آدرس `http://localhost:5173` باز کن و با `ADMIN_USERNAME` / `ADMIN_PASSWORD` وارد شو.

### ۳. ران کردن بات تلگرام

```bash
cd moneymap_bot
python telegram_bot.py
```

(نیاز به متغیرهای محیطی مرتبط با توکن بات و غیره دارد — طبق پیکربندی موجود در ابتدای `telegram_bot.py`.)

## بیلد برای پروداکشن

برای دیپلوی، فقط یک سرویس (Flask) کافی است؛ فرانت‌اند React را build می‌کنیم و Flask همان فایل‌های build‌شده را serve می‌کند:

```bash
cd minimapa
npm install
npm run build
```

این دستور پوشه `minimapa/dist/` را می‌سازد. بعد از آن، اجرای:

```bash
cd ../moneymap_bot
python admin_panel.py
```

هم API (زیر مسیر `/api/*`) و هم رابط کاربری React (از `minimapa/dist/`) را روی همان پورت (پیش‌فرض `8000`) serve می‌کند — دیگر نیازی به اجرای Vite نیست.

اگر `minimapa/dist/` هنوز build نشده باشد، باز کردن آدرس ریشه یک پیام راهنما نشان می‌دهد که بگوید API بالاست ولی فرانت‌اند build نشده.

## ساختار دیتابیس

دیتابیس SQLite به‌صورت خودکار (در اولین اجرای `db.py`، چه از طریق بات و چه از طریق پنل) با جدول‌های زیر ساخته می‌شود:

- `users` — کاربران ثبت‌نام‌شده در بات
- `vip_members` — اعضای دارای اشتراک VIP (با تاریخ انقضا و پرچم‌های یادآوری)
- `analyses` — متن و عکس تحلیل هفتگی هر دارایی (طلا/دلار/بیتکوین)
- `settings` — تنظیمات کلی مثل قیمت و مدت اشتراک VIP (قابل تغییر از صفحه «تنظیمات» در پنل)

تغییر قیمت/مدت اشتراک از پنل، بلادرنگ روی منطق محاسبه‌ی بات (`telegram_bot.py`) هم اعمال می‌شود — چون هر دو از همان جدول `settings` می‌خوانند.
